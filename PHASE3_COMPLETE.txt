═══════════════════════════════════════════════════════════════════
📋 WHAT WAS BUILT
═══════════════════════════════════════════════════════════════════

The REASSIGN workflow allows admins to change an intern's NEXT unit
assignment BEFORE movement is confirmed. This fixes bad fits, staffing
changes, and wrong automated placements without affecting the current
active assignment.


═══════════════════════════════════════════════════════════════════
🎯 KEY FEATURES
═══════════════════════════════════════════════════════════════════

✅ Show only valid units (current unit excluded)
✅ Prevent duplicate rotations
✅ Update next assignment immediately
✅ Dashboard refreshes without page reload
✅ Complete history logging for audit trail
✅ Comprehensive debugging output
✅ Protect current assignment (never changes)
✅ Keep status as awaiting_confirmation (no auto-activation)


═══════════════════════════════════════════════════════════════════
📝 USER FLOW
═══════════════════════════════════════════════════════════════════

1. Admin views Dashboard
2. Finds "Awaiting Confirmation" section
3. Sees intern with current and next unit
4. Clicks [Reassign] button
5. Modal opens with unit selection
6. Admin picks new unit (current excluded)
7. Clicks [Reassign] to confirm
8. Dashboard updates immediately
9. New unit displayed on card
10. Activity log created for audit

🎓 HOW IT WORKS - COMPLETE EXAMPLE
═══════════════════════════════════════════════════════════════════

Scenario: Admin realizes John Doe (Cardiology → Neurology) is a bad
fit and should go to Orthopedics instead.

Timeline:
  Day 20: John finishes Cardiology (duration exceeded)
  → Status: awaiting_confirmation (next: Neurology)
  → Admin sees card on Dashboard

  Admin clicks [Reassign]:
  → ReassignNextModal opens
  → Shows: "Change John Doe's unit from Neurology"
  → Dropdown excludes Cardiology (current)
  → Admin selects Orthopedics

  Admin clicks [Reassign]:
  → API call: POST /api/rotations/{internId}/reassign-next
  → Service validates and updates rotation
  → Activity log created
  → React Query invalidates cache
  → Dashboard refreshes
  → Card now shows: Next: Orthopedics

  Day 30: Admin reviews, clicks [Accept]
  → Cardiology marked complete
  → Orthopedics marked active
  → John moves successfully
