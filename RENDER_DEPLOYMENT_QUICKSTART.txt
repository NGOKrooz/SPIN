┌──────────────────────────────────────────────────────────────────────────────┐
│ STEP 1: Configure Environment Variables in Render Dashboard                 │
└──────────────────────────────────────────────────────────────────────────────┘

Go to: Dashboard → Your Service → Environment

Add these variables:

┌─────────────────────┬──────────────────────────────────────────────────────┐
│ Variable            │ Value                                                │
├─────────────────────┼──────────────────────────────────────────────────────┤
│ DATABASE_URL        │ mongodb+srv://user:pass@cluster.mongodb.net/db       │
│                     │ (Get from MongoDB Atlas → Connect → Connect App)     │
├─────────────────────┼──────────────────────────────────────────────────────┤
│ NODE_ENV            │ production                                           │
├─────────────────────┼──────────────────────────────────────────────────────┤
│ ADMIN_PASSWORD      │ [your-secure-password]                               │
│                     │ (Use password generator - min 16 chars)              │
└─────────────────────┴──────────────────────────────────────────────────────┘

⚠️  CRITICAL: Use MongoDB Atlas connection string with proper authentication!

Example DATABASE_URL format:
mongodb+srv://spinuser:password123@cluster0.xxxxx.mongodb.net/spin_db?retryWrites=true&w=majority

If password contains special characters, URL-encode them:
  @ → %40
  # → %23
  : → %3A
  % → %25

│ STEP 5: Test Deployment                                                     │
└──────────────────────────────────────────────────────────────────────────────┘

1. Health Check:

2. Create a Unit (replace YOUR_PASSWORD):
   curl -X POST https://your-app.onrender.com/api/units \
     -H "Content-Type: application/json" \
     -H "x-admin-key: YOUR_PASSWORD" \
     -d '{"name":"Cardiology","duration_days":30,"workload":"Medium"}'

3. Verify in Frontend:
   https://your-app.onrender.com
   
   - Login with admin password
   - Should see Cardiology unit

┌──────────────────────────────────────────────────────────────────────────────┐
│ TROUBLESHOOTING                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

ERROR: "DATABASE_URL is not configured"
→ Add DATABASE_URL in Render environment variables
→ Redeploy after adding

ERROR: "password authentication failed"
→ Check password in DATABASE_URL is correct
→ Ensure special characters are URL-encoded
→ Verify user exists in Supabase

ERROR: "ENETUNREACH"
→ IPv6 issue (already handled in code)
→ Check Supabase service status
→ Wait 30 seconds and check logs again

ERROR: Build fails
→ Check build logs for specific error
→ Ensure package.json is valid
→ Verify Prisma schema is correct

ERROR: "Route not found"
→ Wait for full server startup (can take 30-60s)
→ Check logs for "All routes loaded successfully"
