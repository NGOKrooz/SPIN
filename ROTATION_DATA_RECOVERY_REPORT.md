# Rotation Data Recovery Report

This report is generated by the recovery audit script:

```bash
npm run audit-rotation-integrity
```

## Purpose

- Identify assignment and rotation corruption across intern histories
- Detect missing unit references, invalid dates, duplicated active assignments, orphaned unit IDs, and invalid statuses
- Provide a structured starting point for recovery and validation

## Summary

The full report is generated when the audit script runs. It includes:

- Total assignments per intern
- Active assignments
- Completed assignments
- Assignments missing unit references
- Assignments missing dates
- Duplicate active assignments
- Orphaned unit IDs
- Invalid statuses

## Next Steps

1. Run `npm run audit-rotation-integrity`
2. Review the generated table and issues
3. Use the results to recover corrupted assignments and verify restored unit relationships
